# Beagle Wind Game

## 系统设计

### 静态资源管理

#### GameNode（游戏机管理）

- 游戏机基本信息管理
  - 设备ID、名称、型号
  - 硬件配置信息
  - 网络状态
  - 地理位置
- 游戏机状态监控
  - 运行状态
  - 资源使用情况
  - 在线状态
- 游戏机操作管理
  - 开机/关机
  - 重启
  - 系统更新
  - 远程维护

#### GamePlatform（游戏平台管理）

- 平台信息管理
  - 平台ID、名称、版本
  - 平台类型（PC、主机、移动端等）
  - 平台特性
- 平台配置管理
  - 系统要求
  - 运行环境配置
  - 网络配置
- 平台版本控制
  - 版本发布
  - 更新管理
  - 回滚机制

#### GameCard（游戏卡片管理）

- 游戏信息管理
  - 游戏ID、名称、类型
  - 游戏描述、封面
  - 游戏分类、标签
- 游戏资源管理
  - 游戏文件
  - 更新包
  - 补丁包
- 游戏配置管理
  - 运行参数
  - 游戏设置
  - 权限控制

### 状态资源管理

#### GameInstance（游戏实例管理）

- 实例生命周期管理
  - 创建实例
  - 启动/停止
  - 暂停/恢复
  - 销毁实例
- 实例状态监控
  - 运行状态
  - 资源占用
  - 性能指标
- 实例数据管理
  - 存档管理
  - 配置同步
  - 数据备份

## 技术架构

### 后端架构

- 微服务架构
- RESTful API
- WebSocket 实时通信
- 数据库设计
  - 原型阶段使用 SQLite 作为本地数据库
  - 静态数据使用 YAML 配置文件管理
  - 提供数据导入工具，支持从 YAML 生成 SQLite 数据库
  - 后续可平滑迁移到其他数据库系统

### 前端架构

- 响应式设计
- 组件化开发
- 状态管理
- UI/UX 设计

### 部署架构

- 容器化部署
- 负载均衡
- 高可用设计
- 灾备方案
