- HTTP 客户端：Axios

## 开发规范

### 1. 命名规范

- 组件名：PascalCase
- 文件名：kebab-case
- 变量名：camelCase
- 常量名：UPPER_CASE

### 2. 目录规范

- 页面组件放在 views 目录
- 公共组件放在 components 目录
- API 接口放在 api 目录
- 工具函数放在 utils 目录

### 3. 组件规范

- 使用 Composition API
- 使用 TypeScript 类型注解
- 组件属性使用 Props 类型定义
- 事件使用 Emit 类型定义

### 4. 样式规范

- 使用 SCSS 预处理器
- 使用 BEM 命名规范
- 组件样式使用 scoped
- 全局样式放在 assets 目录

## 注意事项

1. 代码提交前进行代码格式化
2. 保持组件单一职责
3. 合理使用 TypeScript 类型
4. 注意性能优化
5. 保持代码可维护性
